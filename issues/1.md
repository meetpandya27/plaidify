## Overview
Build an embeddable UI widget (like Plaid Link) that app developers can drop into their frontend. The widget handles the entire connection flow — site selection, credential entry, MFA, progress — in a secure iframe, so credentials never touch the developer's backend.

## The Problem
Currently, developers must build their own frontend to interact with Plaidify's API. This is a high barrier to adoption. Plaid's biggest UX advantage is Plaid Link — a drop-in widget that "just works." Plaidify needs the same.

## Requirements

### React Component
```jsx
<PlaidifyLink
  serverUrl="http://localhost:8000"
  apiKey="your-jwt-token"
  onSuccess={(result) => console.log(result.data)}
  onError={(err) => console.error(err)}
  onMFA={(challenge) => { /* optional custom MFA handling */ }}
  onClose={() => { /* user closed widget */ }}
  theme={{ primaryColor: '#22c55e', borderRadius: '12px' }}
  sites={['greengrid_energy', 'demo_bank']}  // optional: restrict to specific sites
/>
```

### Vanilla JS Version
```html
<script src="https://cdn.plaidify.dev/link.js"></script>
<script>
  PlaidifyLink.open({
    serverUrl: 'http://localhost:8000',
    apiKey: 'your-jwt-token',
    onSuccess: (result) => console.log(result),
    onError: (err) => console.error(err),
  });
</script>
```

### Multi-Step Flow
1. **Site Picker** — search/browse available blueprints
2. **Credential Entry** — username/password form (submitted directly to Plaidify, not developer backend)
3. **Progress** — loading state with step indicator
4. **MFA Challenge** — OTP input, push notification wait, security question
5. **Success** — data summary + close
6. **Error** — friendly error message + retry option

### Security
- Credentials entered in the widget go directly to Plaidify server via HTTPS
- Developer's JavaScript never has access to credentials
- Optional iframe isolation mode for maximum security
- RSA encryption of credentials in transit (reuse existing ephemeral keypair flow)

### Theming
- CSS custom properties for brand customization
- Light/dark mode support
- Configurable: colors, border radius, fonts, logo
- Responsive: works on mobile and desktop

### Packaging
- React: `npm install @plaidify/link-react`
- Vanilla JS: CDN script tag or `npm install @plaidify/link`
- Tree-shakeable, ESM + CJS output
- Target: < 30KB gzipped
